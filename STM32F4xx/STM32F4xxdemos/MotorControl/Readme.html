Description

Following Motor Control examples have been developped with MATLAB R2013b using STM3210B-MCKIT replacing STM32F10-128K-EVAL board (which is in the kit) with STM3240G-EVAL board.

Please refer to STM3210B-MCKIT documentation for jumpers and hardware settings.

For following examples, please set:

  • Power supply:

Set Jumpers as described in "Shinano 24V PMSM motor (provided)".

  • W1: <35V only
  • W6: Present
  • W7: Present and set on DEFAULT
  • W8: Present
  • W9: Present
  • W11: Present
  • W12: Not present
  • W13: Not present
  • W14: Not present
  • W15: Not present
  • W16: Present and set on REVERSE
  • W17: Present
  • W18: Present
  • W19: Present

Repositories :

  • FOC : Field-Oriented Control

FOC - Field-Oriented Control

It is based on a Park transformation for sinuzoidal with encoder example.

It computes 3 phases output currents in order to maximize perpendicular field to the permanent magnet.

This example gives opportunity to :

  • do pure simulation = PC simulation
  • do Processor In the Loop (PIL) simulation
  • build and run project on motor kit hardware.

Pure simulation, PIL simulation and Motor Control Kit application are based on the same reference model (PMSM_FOC_Compute.slx).

This model computes output current to be applied to motor depending on speed input and loop back values.

PMSM_FOC model has been developped and must be used for both Pure simulation and PIL simulation

top

----------------------------------------------------------------------

To do Pure simulation you must set :

  1. PMSM_FOC_Compute block parameter>simulation mode to "Normal"
  2. PMSM_FOC model simulation mode parameter to "Normal"

Then, set PMSM_FOC model "Simulation stop time" parameter to the desired simulation time (default is 0.8) and start simulation clicking "Run" button.

top

----------------------------------------------------------------------

To do PIL simulation you must set :

  1. PMSM_FOC_Compute block parameter>simulation mode to "Processor-in-the-loop (PIL)"
  2. PMSM_FOC model simulation mode parameter to "Processor-in-the-loop (PIL)"

Then, set PMSM_FOC model "Simulation stop time" parameter to the desired PIL simulation time (default is 0.2) and start simulation clicking "Run" button.

Note that PIL takes long time regarding "pure simulation" time because of Serial speed communication (115200bds) link between PC and ST Evaluation board.

top

----------------------------------------------------------------------

To generate STM32F4 target application based on peripherals toolset:

Open PMSM_FOC_Encoder_ST model and set:

  1. PMSM_FOC_Compute block parameter>simulation mode to "Normal"
  2. PMSM_FOC_Encoder_ST model simulation mode parameter to "Normal"
  3. Build STM32F4 project clicking "Build model" button.

HallSensor

It is based on trapezoidal with Hall sensor example. It computes 3 phases output currents with trapezoidal truth table, depending on Hall sensor information.

This example gives opportunity to :

  • do pure simulation (PC simulation) for Trapezoidal open loop example
  • do pure simulation (PC simulation) for Trapezoidal torque control example
  • build and run project on motor kit hardware.

Open PMSM_Trapezoidal_OpenLoop.slx and PMSM_Trapezoidal.slx examples models are used for PC simulation. Open model and "Run" it to do PC simulation. As Hall sensors give +/-30° rotor position, you can see that computed field is not always perpendicular to permanent magnet.

Open PMSM_Trapezoidal_ToolBoxST.slx and press "Build" button to generate, build and download STM32F4 application to motor control kit.

  1. Click Add with subfolders... button of "Set Path" window.
  2. Select STM32F4xx folder (It contains this Readme file)
  3. Click OK
  4. Click Save button and close "Set Path" window

PMSM

It is based on MATLAB default Permanent Magnet Synchronous Machine System Test Bench example and used for *P*rocessor *I*n the *L*oop demonstration. rtwdemo_pmsmfoc_system_ST.slx is top level model including rtwdemo_pmsmfoc_ST.slx reference model.

To run PIL simulation, you must set:

  1. rtwdemo_pmsmfoc_ST block parameter>simulation mode to "Processor-in-the-loop (PIL)"
  2. rtwdemo_pmsmfoc_system_ST model simulation mode parameter to "Processor-in-the-loop (PIL)"

Then, set rtwdemo_pmsmfoc_system_ST model "Simulation stop time" parameter to the desired PIL simulation time and start simulation clicking "Run" button.

Note that PIL takes long time regarding "pure simulation" time because of Serial speed communication (115200bds) link between PC and ST Evaluation board.

Util

It is utility repository including MATLAB script used to draw motor rotor rotation and generated field during simulation.
Fields are proportional to phases current.
It includes arrow.m script developped by Dr. Erik A. Johnson JohnsonE@usc.edu
